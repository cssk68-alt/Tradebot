This conversation traces the implementation of two major architectural upgrades to a Polymarket prediction-market trading bot:

Phase 1 (Pre-session): Probabilistic Pattern Engine v2 — A complete rewrite of the pattern learning system from binary hard rules (p-value gates, is_hard_rule, _promote_to_hard_rule) to a continuous probabilistic framework (statistical_significance_score, stability_score, decay_adjusted_confidence, constraint_strength). This was completed before the current session.

Phase 2 (This session, Sequence 6+): For-The-Future Learner Meta-Learning LLM — A non-intrusive observer LLM agent that analyzes completed trade batches post-execution, generating structured insights for the dashboard without affecting live decision logic.

The session focused on building a clean, architecturally separated meta-learning system that extends the existing PostmortemAgent pattern but operates on batch cross-trade analysis rather than per-trade lessons.

Key architectural decisions made:

Meta-Learner is STRICTLY observer-only, with no write access to any decision-making state
Existing export_state() -> Brain pattern preserved (dashboard reads brain.meta_insight())
Meta-Learner runs AFTER _after_resolved, after pattern engine and training
Output format mandated by user: insight_summary, confidence_of_insight, category_tags, suggested_future_hypotheses

Known Architectural Design Notes (Not Bugs)
Meta-Learner has no persistence to DB — insights are ephemeral (in-memory + dashboard JSON). If persistence is needed, the MetaInsight model would need a DB table and save_meta_insight()/load_meta_insight() methods on Store.

Meta-Learner requires LLM agent — falls back silently (no error) if no LLM client is available, but never degrades execution. The available check inside run() means it simply skips analysis.

Batch size capped at 20 trades — _build_trade_history() uses resolved[-20:] to keep LLM prompts reasonable. If there are more than 20 recent trades, older ones are excluded.

Min trade threshold = 3 — run() returns None if len(resolved) < 3, as cross-trade pattern detection needs multiple data points.

Pattern engine v2 still has hard_rules in log message — The log message in feedback.py line ~155 says "loaded %d patterns (%d observations, %d hard rules)" — this is cosmetic only (uses ps.get("hard_rules", 0) which will be 0 since v2 removed hard rules). Not a functional bug.
